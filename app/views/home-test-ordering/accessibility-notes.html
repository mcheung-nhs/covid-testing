Accessibility testing notes Order a home test kit

Summary

  1. The site does not closely follow the GOV.UK Design System example code and styling. Thus there are minor inconsistencies in look compared to other government services that use the Design System patterns.
  2. The screens contain no <header> or <main> elements. These landmarks would help screen reader users with navigation.
  3. The URL does not change when moving from screen to screen. Therefore if you refresh the screen at any time, you will be sent back to the start. This could be confusing to the user. Another consequence of this is that the browser 'Back' button would not work as expected. It would take the user back to the last site they had been on (if any) and not to the previous screen. This would again cause confusion.
  4. Check the use of the "Start" style of buttons. According to the Design System, the start button should only be used "for the main call to action on your service's start page". The default button should be used for all other screens.
  5. Use of the reCAPTCHA functionality produces extra "privacy policy" and "terms and conditions" links on the page, with no differentiation between the page links and the reCAPTCHA links. This may cause confusion for screen reader users.

  1. Start page

    • Page with sample HTML: start

    Comments

    1. No <header> or <main> elements on the page. These landmarks would help screen reader users with navigation.
    2. Incorrect use of heading - the text "Use this test as instructed. Only order the number of kits you need - do not order extras." is a <h3>. It is not a header as it does not have any related content beneath it. Screen reader users often rely on navigating by heading so if the structure isn’t hierarchical they may not understand the relationship. It should be marked up as a paragraph or use the warning text component.
    3. Disabled buttons are not recommended by the GOV.UK Design System. Consider changing this to displaying an error if the checkbox is not checked.

    References

    • https://design-system.service.gov.uk/components/warning-text/
    • https://design-system.service.gov.uk/components/button/#disabled-buttons

  3. Enter your details

    • Page with sample HTML: user-details

    Comments

    1. There is no <h1> explaining what the page is about
    2. It is recommended that you ask one question per page
    3. <fieldset> and <legend> elements should be used to group input fields together to improve the screen reader experience
    4. The second input field for "Building and street" has no <label> so is not described correctly by screen readers
    5. On VoiceOver when using the form controls menu, the labels for the National Insurance Number and email address fields are not read out as expected. The form fields are described by the text contained in the (invalid) "placeholder" attribute of the <label> element. This may cause confusion to screen reader users.
    6. When clicking on the 'Continue' button, if there are any errors, then screen readers user are not told. As the page is not reloaded, focus does jump to the top of the page but this is not conveyed to the user. Therefore this would probably cause confusion. Adding and aria-live attribute with a value of "polite" to the error summary box or surrounding container may improve this.
    7. Each of the error messages in the error summary box do not link to anywhere. They should link to the input field that has the error.

    References

    • https://design-system.service.gov.uk/patterns/question-pages/#start-by-asking-one-question-per-page
    • https://www.gov.uk/service-manual/design/form-structure#start-with-one-thing-per-page
    • https://design-system.service.gov.uk/components/fieldset/
    • https://design-system.service.gov.uk/components/text-input/

  9. Order summary

    • Page with sample HTML: order-summary

    Comments

    1. The heading of this page is a h2 when it should be a h1
    2. The "Disclaimer" heading shoud be a h2
    3. The order summary is marked up as a <table> which does not scale well on smaller screen sizes as it causes horizontal scrolling. Consider using a <dl> element to mark up it up.
    4. Recommend adding visibly hidden text for the "Change" links to give more context for screen reader users
    5. The "Change" links do not appear on the links web item rotor on VoiceOver. This is used by screen reader users to find all the links on a page for easier navigation. This is probably due to the links not having a "href" atttribute.
    6. Disabled buttons are not recommended by the GOV.UK Design System. Consider changing this to displaying an error if the checkbox is not checked.
    7. The links in the reCAPTCHA widget are being navigated to sometime after the order summary which may be confusing to screen reader users. Where they are read seems to depend on the size of the browser window.

    References

    • https://design-system.service.gov.uk/patterns/check-answers/
    • https://design-system.service.gov.uk/components/table/
