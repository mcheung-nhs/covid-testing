  2. Have you booked your courier?

    Tested link: book-courier
    Page with sample HTML: book-courier
    Error page: book-courier-error

    Comments

    1. Missing <legend> element which is a requirement inside the <fieldset> element. This can be added and made a heading.
    2. Consider associating the hint text with the <fieldset> to better describe it for screen readers
    3. For the error page:
      1. Recommend adding "Error:" to the page title
      2. Ensure that the error message in the summary box links to the first radio button and not to the error message
      3. Recommend the error message be more relevant e.g. "Select if you have booked your courier"

  9. What's your date of birth?

    Tested link: age
    Page with sample HTML: age
    Error page: age-error

    Comments

    1. Empty <legend> element which is a requirement inside the <fieldset> element. This can be added and made a heading.
    2. Consider associating the hint text with the <fieldset> to improve the screen reader experience.
    3. Recommend adding the autocomplete attribute to the input fields. This lets browsers autofill the information on a user’s behalf if they’ve entered it previously. See reference link below.

    References

    • https://design-system.service.gov.uk/components/date-input/
    • https://design-system.service.gov.uk/components/error-summary/
