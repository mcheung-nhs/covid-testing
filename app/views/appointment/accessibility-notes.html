  1. Start page

    Tested link: appointment
    Page with sample HTML: start

    Comments

    1. Incorrect heading structure - the headings below the <h1> are <h4> when they should be <h2>. Screen reader users often rely on navigating by heading so if the structure isn’t hierarchical they may not understand the relationship.

  2. What's your invitation code?

    Tested link: validate-code
    Page with sample HTML: validate-code

    Comments

    1. Hint text is not correctly associated with the <fieldset>. Correcting this improves the screen reader experience.
    2. Conside using 1 field rather than 4 separate ones which can cause confusion due to layout issues when zoomed in on various devices. This would need to be tested along with the invitation code that is sent via text message to determine the best format.

    Example code

    
<fieldset class="govuk-fieldset" aria-describedby="code-hint">
  <legend class="govuk-fieldset__legend govuk-fieldset__legend--s">Enter your invitation code</legend>
  <span class="govuk-hint" id="code-hint">This is the 16 digit code you received in the text message we sent you.</span>
  
  <!-- input fields -->

</fieldset>

  4. What's your vehicle registration number?

    Tested link: vehicle-registration
    Page with sample HTML: vehicle-registration

    Comments

    1. Incorrect use of <fieldset> and <legend> elements. There is only 1 input field so there is no need to use a <fieldset>.
    2. There are 2 <label> elements to describe the 1 input field.
    3. Hint text is not correctly associated with the input field. However, it could be argued that because there is a lot of hint text, then this creates extra 'noise' that screen readers will read out so there isn't a need to link it to the input. This may need user testing.

    References

    • https://design-system.service.gov.uk/components/fieldset/
    • https://design-system.service.gov.uk/components/text-input/

    Example code

    
<form>
  <div class="govuk-form-group">
    <label class="govuk-label" for="vehicle-registration">
      Vehicle registration number (number plate). For example, CU57ABC
    </label>

      <p class="govuk-hint">This must be the number plate of the vehicle you’ll use to visit the test site. It does not have to be registered in your name, just as long as it’s the vehicle you turn up in. This can be a car or a small van.</p>
      <p class="govuk-hint">You cannot turn up on foot or in a taxi.</p>
    </div>

  6. What's your date of birth?

    Tested link: age
    Page with sample HTML: age

    Comments

    1. Empty <legend> element which is a requirement inside the <fieldset> element. This can be added and made a heading.
    2. Hint text is not correctly associated with the <fieldset>. Correcting this improves the screen reader experience.
    3. Recommend adding the autocomplete attribute to the input fields. This lets browsers autofill the information on a user’s behalf if they’ve entered it previously. See reference link below.

    References

    • https://design-system.service.gov.uk/components/date-input/

    Example code

    
<form>
  <div class="govuk-form-group">
    <fieldset class="govuk-fieldset" role="group" aria-describedby="dob-hint">
      <legend class="govuk-fieldset__legend govuk-fieldset__legend--xl">
        <h1 class="govuk-fieldset__heading">
          What's your date of birth?
        </h1>
      </legend>
      <span id="dob-hint" class="govuk-hint">
        For example, 31 3 1980
      </span>
      <div class="govuk-date-input" id="dob">
        <div class="govuk-date-input__item">
          <div class="govuk-form-group">
            <label class="govuk-label govuk-date-input__label" for="date-day">
              Day
            </label>
            <input class="govuk-input govuk-date-input__input govuk-input--width-2" id="date-day" name="day" type="text" autocomplete="bday-day" pattern="[0-9]*" inputmode="numeric">
          </div>
        </div>
        <div class="govuk-date-input__item">
          <div class="govuk-form-group">
            <label class="govuk-label govuk-date-input__label" for="date-month">
              Month
            </label>
            <input class="govuk-input govuk-date-input__input govuk-input--width-2" id="date-month" name="month" type="text" autocomplete="bday-month" pattern="[0-9]*" inputmode="numeric">
          </div>
        </div>
        <div class="govuk-date-input__item">
          <div class="govuk-form-group">
            <label class="govuk-label govuk-date-input__label" for="date-year">
              Year
            </label>
            <input class="govuk-input govuk-date-input__input govuk-input--width-4" id="date-year" name="year" type="text" autocomplete="bday-year" pattern="[0-9]*" inputmode="numeric">
          </div>
        </div>
      </div>
    </fieldset>
  </div>
  <button class="govuk-button">Continue</button>
</form>
